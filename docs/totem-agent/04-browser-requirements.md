# Browser-Specific Requirements

## Overview
Totem must support three browser platforms:
1. **Chrome/Edge** - Manifest V3, chromium API
2. **Firefox** - WebExtension API, manifest v2/v3 hybrid
3. **Safari** - App Extension (Safari 14+), native bridge

Each platform has unique manifest formats, API differences, viewport constraints, and packaging requirements.

## Manifest Formats

### Chrome/Edge (Manifest V3)
**File**: `manifest.json`

```json
{
  "manifest_version": 3,
  "name": "Totem Wallet",
  "version": "1.0.0",
  "description": "Quantum-resistant wallet for Minima blockchain",
  
  "action": {
    "default_popup": "popup.html",
    "default_icon": {
      "16": "icons/icon-16.png",
      "32": "icons/icon-32.png",
      "48": "icons/icon-48.png",
      "128": "icons/icon-128.png"
    }
  },
  
  "background": {
    "service_worker": "background.js",
    "type": "module"
  },
  
  "content_scripts": [
    {
      "matches": ["<all_urls>"],
      "js": ["content-script.js"],
      "run_at": "document_start"
    }
  ],
  
  "permissions": [
    "storage",
    "unlimitedStorage",
    "activeTab"
  ],
  
  "host_permissions": [
    "https://rpc.axia.to/*"
  ],
  
  "web_accessible_resources": [
    {
      "resources": ["provider.js"],
      "matches": ["<all_urls>"]
    }
  ],
  
  "content_security_policy": {
    "extension_pages": "script-src 'self'; object-src 'self'"
  }
}
```

**Key Differences from MV2**:
- ❌ No persistent background pages → Service workers only
- ❌ No `browser_action` → Use `action` instead
- ✅ `host_permissions` separate from `permissions`
- ✅ `web_accessible_resources` requires `matches` array

### Firefox (WebExtension - Manifest V2/V3 Hybrid)
**File**: `manifest.firefox.json`

```json
{
  "manifest_version": 2,
  "name": "Totem Wallet",
  "version": "1.0.0",
  "description": "Quantum-resistant wallet for Minima blockchain",
  
  "browser_action": {
    "default_popup": "popup.html",
    "default_icon": {
      "16": "icons/icon-16.png",
      "32": "icons/icon-32.png",
      "48": "icons/icon-48.png",
      "128": "icons/icon-128.png"
    },
    "browser_style": false
  },
  
  "background": {
    "scripts": ["background.js"],
    "persistent": false
  },
  
  "content_scripts": [
    {
      "matches": ["<all_urls>"],
      "js": ["content-script.js"],
      "run_at": "document_start"
    }
  ],
  
  "permissions": [
    "storage",
    "unlimitedStorage",
    "activeTab",
    "https://rpc.axia.to/*"
  ],
  
  "web_accessible_resources": ["provider.js"],
  
  "content_security_policy": "script-src 'self'; object-src 'self'",
  
  "browser_specific_settings": {
    "gecko": {
      "id": "{totem-wallet@axia.to}",
      "strict_min_version": "109.0"
    }
  }
}
```

**Key Differences**:
- ✅ Still supports MV2 (recommended for stability)
- ✅ `browser_action` instead of `action`
- ✅ `background.scripts` array instead of `service_worker`
- ✅ Permissions include host permissions (no separate `host_permissions`)
- ✅ Requires `browser_specific_settings.gecko.id` for signing

### Safari (App Extension - Manifest Not Used)
**File**: `Info.plist` (generated by Xcode)

Safari extensions are **native Swift apps** with an embedded web extension bundle. The manifest is replaced by `Info.plist`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN">
<plist version="1.0">
<dict>
  <key>NSExtension</key>
  <dict>
    <key>NSExtensionPointIdentifier</key>
    <string>com.apple.Safari.extension</string>
    
    <key>NSExtensionPrincipalClass</key>
    <string>$(PRODUCT_MODULE_NAME).SafariWebExtensionHandler</string>
    
    <key>SFSafariWebsiteAccess</key>
    <dict>
      <key>Level</key>
      <string>All</string>
    </dict>
    
    <key>SFSafariContentScript</key>
    <array>
      <dict>
        <key>Script</key>
        <string>content-script.js</string>
      </dict>
    </array>
  </dict>
  
  <key>CFBundleDisplayName</key>
  <string>Totem Wallet</string>
  
  <key>CFBundleIdentifier</key>
  <string>to.axia.totem</string>
  
  <key>CFBundleVersion</key>
  <string>1.0.0</string>
</dict>
</plist>
```

**Build Requirements**:
- Xcode 13+ (macOS only)
- Apple Developer account ($99/year for signing)
- App Sandbox entitlements
- Native messaging bridge (Swift ↔ JavaScript)

## API Differences

### Chrome vs Browser Namespace
**Problem**: Chrome uses `chrome.*` API, Firefox uses `browser.*` (Promises).

**Solution**: Polyfill for cross-browser compatibility

```typescript
// packages/totem-extension/src/core/polyfills/browser.ts

declare global {
  interface Window {
    browser: typeof chrome;
  }
}

// Firefox: browser.* is native and Promise-based
// Chrome: chrome.* uses callbacks, browser.* polyfill available

if (typeof chrome !== 'undefined' && !chrome.runtime.onMessage.hasListeners) {
  // Chrome MV3: Use chrome.* API
  globalThis.browser = chrome;
} else if (typeof browser !== 'undefined') {
  // Firefox: browser.* is already defined
  globalThis.browser = browser;
}

// Usage: Always use `browser.*` in code
await browser.storage.local.set({ key: 'value' });
```

### Storage API
#### Chrome
```typescript
// Callback-based (MV2)
chrome.storage.local.get(['key'], (result) => {
  console.log(result.key);
});

// Promise-based (MV3 with polyfill)
const result = await chrome.storage.local.get(['key']);
console.log(result.key);
```

#### Firefox
```typescript
// Always Promise-based
const result = await browser.storage.local.get(['key']);
console.log(result.key);
```

**Best Practice**: Use `browser.*` with Promises everywhere.

### Messaging API
#### Chrome
```typescript
// Send message
chrome.runtime.sendMessage({ method: 'test' }, (response) => {
  console.log(response);
});

// Listen
chrome.runtime.onMessage.addListener((message, sender, sendResponse) => {
  sendResponse({ ok: true });
  return true; // Keep channel open for async
});
```

#### Firefox
```typescript
// Send message
const response = await browser.runtime.sendMessage({ method: 'test' });
console.log(response);

// Listen
browser.runtime.onMessage.addListener((message, sender) => {
  return Promise.resolve({ ok: true });
});
```

**Unified Pattern**:
```typescript
// Works in both
browser.runtime.onMessage.addListener((message, sender, sendResponse) => {
  handleMessage(message).then(sendResponse);
  return true; // Required for Chrome async responses
});
```

### Tabs API
#### Chrome
```typescript
chrome.tabs.query({ active: true, currentWindow: true }, (tabs) => {
  const activeTab = tabs[0];
});
```

#### Firefox
```typescript
const tabs = await browser.tabs.query({ active: true, currentWindow: true });
const activeTab = tabs[0];
```

### Safari
Safari uses the **same WebExtension API** as Firefox/Chrome with minor differences:

```swift
// Swift handler for native bridge
class SafariWebExtensionHandler: NSObject, NSExtensionRequestHandling {
  func beginRequest(with context: NSExtensionContext) {
    // Handle native requests from JavaScript
  }
}
```

**Limitations**:
- ❌ No `chrome.declarativeNetRequest` (content blockers use different API)
- ❌ Limited `chrome.webRequest` (use Safari Content Blockers)
- ✅ `browser.storage.local` works (backed by UserDefaults)
- ✅ Messaging works (bridged through native Swift)

## Viewport Specifications

### Chrome/Edge Popup
```css
/* Recommended size */
width: 400px;
height: 600px;

/* Min/Max enforced by browser */
min-width: 25px;
min-height: 25px;
max-width: 800px;
max-height: 600px;
```

**Quirks**:
- Opens in **new window** if detached
- Can be resized by user (detached mode only)
- Closes when losing focus (default behavior)

### Firefox Popup
```css
/* Panel size (smaller than Chrome) */
width: 380px;
height: 580px;

/* Firefox enforces stricter limits */
max-width: 420px;
max-height: 600px;
```

**Quirks**:
- Panel is **anchored to toolbar icon** (cannot detach)
- Auto-closes when clicking outside
- Cannot be resized by user
- **Scrollbars appear if content exceeds height** (avoid at all costs)

**Testing Strategy**: Always test at **380px width** (Firefox constraint)

### Safari Popover
```css
/* macOS Big Sur+ */
width: 420px;
height: 620px;

/* Safari enforces */
max-width: 500px;
max-height: 700px;
```

**Quirks**:
- Popover is **native macOS view** (not HTML)
- Uses Safari's color scheme (light/dark mode)
- Native scrolling (smooth, inertia)
- Can be detached on macOS 12+ (dragged to separate window)

### Responsive Design
**Mobile Not Supported**: Browser extensions are desktop-only. Mobile users should use:
- Progressive Web App (PWA)
- Native iOS/Android app
- Mobile-optimized website

## Build System Differences

### Chrome/Edge Build
```json
// package.json
{
  "scripts": {
    "build:chrome": "WOTS_PARAMSET=v2-spec webpack --config webpack.chrome.js --mode=production",
    "package:chrome": "cd dist/chrome && zip -r ../../totem-chrome-1.0.0.zip ."
  }
}
```

**Output**: `dist/chrome/` directory with:
- `manifest.json` (MV3)
- `popup.html`, `background.js`, `content-script.js`
- `icons/` folder

**Package Format**: `.zip` file uploaded to Chrome Web Store

### Firefox Build
```json
{
  "scripts": {
    "build:firefox": "WOTS_PARAMSET=v2-spec webpack --config webpack.firefox.js --mode=production",
    "package:firefox": "cd dist/firefox && web-ext build --overwrite-dest"
  }
}
```

**Output**: `dist/firefox/` directory with:
- `manifest.json` (MV2)
- Same files as Chrome but with Firefox-specific polyfills

**Package Format**: `.xpi` file (signed by Mozilla)

**Signing Process**:
1. Create account at addons.mozilla.org
2. Generate API credentials
3. Run: `web-ext sign --api-key=<key> --api-secret=<secret>`
4. Receives signed `.xpi` file

### Safari Build
**Requires Xcode Project**:

```bash
# 1. Convert extension to Safari App Extension
xcrun safari-web-extension-converter dist/chrome

# 2. Open in Xcode
open Totem\ Wallet/Totem\ Wallet.xcodeproj

# 3. Build & Archive
# Product → Archive (Xcode menu)

# 4. Distribute to App Store
# Window → Organizer → Distribute App
```

**Output**: `.app` bundle containing:
- Native Swift wrapper
- Embedded web extension
- Code signed with Apple Developer certificate

**Package Format**: `.pkg` installer or App Store submission

**Signing Requirements**:
- Apple Developer account ($99/year)
- Code signing certificate
- App Sandbox entitlements:
  ```xml
  <key>com.apple.security.app-sandbox</key>
  <true/>
  <key>com.apple.security.network.client</key>
  <true/>
  ```

### Feature Detection
```typescript
// Check if running in service worker (Chrome MV3)
const isServiceWorker = typeof importScripts === 'function';

// Check if browser supports webRequest (not Safari)
const hasWebRequest = 'webRequest' in browser;
```

## Common Gotchas

### 1. Service Worker Lifecycle (Chrome MV3)
**Problem**: Service workers terminate after 30 seconds of inactivity.

**Solution**: Store state in `chrome.storage`, not in-memory.

```typescript
// ❌ BAD: Lost when service worker terminates
let currentUser = { address: '0x...' };

// ✅ GOOD: Persisted across restarts
await chrome.storage.local.set({ currentUser: { address: '0x...' } });
```

### 2. CSP (Content Security Policy)
**Problem**: Inline scripts blocked in popup/background.

**Solution**: Use external script files only.

```html
<!-- ❌ BAD -->
<script>console.log('test');</script>

<!-- ✅ GOOD -->
<script src="popup.js"></script>
```

### 3. CORS in Content Scripts
**Problem**: Content scripts inherit page's CSP, blocking API calls.

**Solution**: Proxy all API calls through background script.

```typescript
// content-script.js - Don't fetch directly
// ❌ BAD
const response = await fetch('https://rpc.axia.to/balance');

// ✅ GOOD: Ask background to fetch
const response = await browser.runtime.sendMessage({
  method: 'proxy_fetch',
  url: 'https://rpc.axia.to/balance'
});
```

### 4. Firefox Popup Auto-Close
**Problem**: Firefox panel closes when clicking outside.

**Solution**: Save state before closing, restore on reopen.

```typescript
// popup.tsx
useEffect(() => {
  const saveState = async () => {
    await browser.storage.local.set({ popupState: { tab: activeTab } });
  };
  
  window.addEventListener('blur', saveState);
  return () => window.removeEventListener('blur', saveState);
}, [activeTab]);
```

### 5. Safari Native Bridge Latency
**Problem**: Calls to native Swift layer have ~50ms overhead.

**Solution**: Batch operations, cache aggressively.

```typescript
// ❌ BAD: Multiple native calls
for (let i = 0; i < 100; i++) {
  await browser.storage.local.set({ [`key${i}`]: value });
}

// ✅ GOOD: Single batched call
const batch = {};
for (let i = 0; i < 100; i++) {
  batch[`key${i}`] = value;
}
await browser.storage.local.set(batch);
```

## Browser Support Matrix

| Feature | Chrome 109+ | Firefox 109+ | Safari 14+ |
|---------|-------------|--------------|------------|
| Manifest V3 | ✅ | ⚠️ (MV2 recommended) | ✅ (via converter) |
| Service Workers | ✅ | ⚠️ (background scripts) | ✅ |
| `browser.*` API | ✅ (polyfill) | ✅ (native) | ✅ |
| WebCrypto | ✅ | ✅ | ✅ |
| IndexedDB | ✅ | ✅ | ✅ |
| `storage.local` | ✅ (unlimited) | ✅ (10MB limit) | ✅ (UserDefaults) |
| Content Scripts | ✅ | ✅ | ✅ |
| `webRequest` | ⚠️ (deprecated) | ✅ | ❌ (use Content Blockers) |
| Native Messaging | ✅ | ✅ | ✅ (Swift bridge) |

**Minimum Versions**:
- Chrome/Edge: 109+ (MV3 stable)
- Firefox: 109+ (MV2/MV3 hybrid)
- Safari: 14+ (macOS 11 Big Sur)
